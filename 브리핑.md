## 2. 확정된 결정사항

### 아키텍처
- **데이터 저장**: 예약 데이터는 Firestore에 먼저 저장 → `doPost`로 GAS 호출 → Google Sheets에 복사
- **동기화 방식**: `doPost` 방식 확정 (1분 트리거 REST API 방식은 Firebase IAM 403 오류로 폐기)
- **메일 발송**: GAS `sendConfirmMail`/`sendChangeMail`/`sendCancelMail` — `doPost` 수신 시 동시 발송
- **메일 내 버튼 URL**: `https://tinyurl.com/hicmajors`
- **체크인 목록 기준**: `getSessionStatus`는 CheckIns 시트만 기반 — SessionPreReg에 있는 미체크인자는 포함하지 않음
- **사전예약자 체크인**: 체크인 시점의 세션부터 이후 전체 세션을 참여로 기록 (당일방문자와 동일)

## 4. 활성 제약사항

### 개발 운영 규칙
- `clasp push`: Claude가 직접 수행
- `git push`: Claude가 직접 수행
- **index.html 수정 시**: `git push` → GitHub Pages 자동 반영 (GAS 재배포 불필요)
- **staff.html / admin.html 수정 시**: `clasp push` + GAS 재배포 필요
  - 재배포 경로: 배포 → 배포 관리 → 연필(✏️) → 새 버전 선택 → 배포
- **Code.js 수정 시**: `clasp push` + **GAS 재배포 필요** (doPost 등 웹앱 엔드포인트 변경 포함)
  - 재배포 후 반드시 `clasp deployments`로 최신 배포 ID 확인 → `index.html` GAS_POST_URL / GAS_CONFIG_URL과 비교 → 다르면 교체 후 `git push`

### 예약 비즈니스 규칙 (5가지)
1. 동일 부스는 한 학번에 하나의 시간대만 예약 가능
2. 기존 예약이 있으면 신규 제출 차단 → 변경 모드로 유도
3. 동일 시간대에 여러 부스 중복 예약 불가
4. 라이프디자인센터 부스는 한양인터칼리지학부만 예약 가능
5. 설명회는 프로그램당 1회만 등록 가능

### 기술 제약
- Firebase 보안규칙 문법: `allow read: if true;` — `if` 키워드 필수 (없으면 `^` 문법오류)
- `google.script.run`은 GAS 웹앱으로 서빙된 페이지에서만 동작 — GitHub Pages에서는 사용 불가
- 비밀번호: AdminUsers 시트에서 관리 (기본값 `hic2025`)
